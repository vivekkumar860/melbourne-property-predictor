Melbourne Property Price Prediction Project

This project is a Streamlit application that predicts property prices in Melbourne using a neural network model built with TensorFlow.

Technologies Used

  • Streamlit: Used for building the web application interface.
  • Pandas & NumPy: For data manipulation and numerical operations.
  • Scikit-learn: For machine learning algorithms and data preprocessing.
  • TensorFlow: For building and training the neural network model.
  • Matplotlib & Seaborn: For data visualization.
  • Plotly: For interactive plots.
  • Folium: For creating interactive maps.
  • Pillow: For image processing.
  • Requests: For making HTTP requests.
  • Watchdog: For file system monitoring to improve Streamlit performance.

Features

  • Property Price Prediction: Utilizes a neural network model to predict property prices based on various features such as location, size, and amenities.
  • Interactive Visualizations: Provides interactive charts and maps to visualize property data and trends.
  • FAQ Section: Includes a comprehensive FAQ section to address common questions about the application and its features.
  • About the Model: Detailed explanation of the model's architecture, training process, and limitations to help users understand the prediction mechanism.

Installation

To install the required dependencies, run:

pip install -r requirements.txt

Running the Application

To run the application, use the following command:

python3 -m streamlit run app.py

Access the application at http://localhost:8501.

Deploying the Project Online

There are several ways to deploy your Streamlit application online:

  • Streamlit Sharing: Streamlit offers a free hosting service called Streamlit Sharing. You can deploy your app by pushing your code to a GitHub repository and connecting it to Streamlit Sharing. Follow these steps:
    1. Push your code to a GitHub repository.
    2. Sign up for Streamlit Sharing and connect your GitHub account.
    3. Select the repository and branch to deploy.
    4. Streamlit will automatically deploy your app and provide a URL.
  • Heroku: Deploy your app on Heroku by creating a Procfile and requirements.txt file, then pushing your code to a Heroku app. Steps:
    1. Create a Procfile with the command to run your app: web: streamlit run app.py.
    2. Ensure your requirements.txt is up to date.
    3. Install the Heroku CLI and log in.
    4. Create a new Heroku app and push your code: git push heroku master.
  • AWS Elastic Beanstalk: Use AWS Elastic Beanstalk to deploy your application. You'll need to create a configuration file and set up an environment on AWS. Steps:
    1. Create a .ebextensions directory with a configuration file to set up the environment.
    2. Install the AWS CLI and configure it with your credentials.
    3. Create an Elastic Beanstalk environment and deploy your application.
  • Google Cloud Run: Deploy your app on Google Cloud Run by containerizing your application and pushing it to Google Container Registry. Steps:
    1. Create a Dockerfile to containerize your application.
    2. Build and push the Docker image to Google Container Registry.
    3. Deploy the container to Google Cloud Run.
  • Docker: Containerize your application using Docker and deploy it on any cloud platform that supports Docker containers. Steps:
    1. Create a Dockerfile to define your application's environment.
    2. Build the Docker image and push it to a container registry.
    3. Deploy the container on your chosen cloud platform.

Each method has its own set of steps and requirements. Choose the one that best fits your needs and follow the detailed guides provided by each platform.
